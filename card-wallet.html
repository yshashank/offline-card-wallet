How to update this wallet

Monthly update cadence (~10 minutes)

  1. Open the file and check the summary strip at the top — it flags expired cards, cards expiring within 60 days, and points balances more than 90 days old ("update due").
  2. Tap Edit (top right). Edit mode stays off by default so day-to-day browsing can't change anything.
  3. For each flagged card: open it → Edit card → refresh the points balance and set Points as on to today. Add any new cards, delete dead programs.
  4. Press Save file (it turns red whenever there are unsaved changes). A fresh copy of this file downloads, named after the wallet (e.g. family-card-wallet.html), with all your data inside it.
  5. Replace the old copy in Google Drive / NAS with the downloaded file. This step is the actual save — edits live only in the browser until you do it.
  6. Optional: Export CSV for an offline Excel-readable snapshot. Card numbers are deliberately wrapped as ="…" in the CSV — that formula wrapper is what stops Excel mangling long numbers into scientific notation, so leave it as-is.

Rollback: Google Drive keeps version history on the file — right-click → Manage versions. The NAS mirror is your second line.

Day-to-day rules

  • Treat the file as read-only outside the monthly window. Tap a card to view details; use Copy next to the number at counters, or tap the number itself (or the QR button) for a large scannable QR code with the number in big type beneath it — note some store scanners only read 1D barcodes, in which case show staff the large number.
  • Points chips age visibly: bright green = updated within 30 days, muted = 31–90 days, amber "update due" = older than 90 days.
  • Loyalty and membership numbers only. Never store payment card numbers, PINs, passwords, CVVs, or OTP seeds here — those belong in the password manager.
  • Share via Drive permissions to specific family accounts only; never via public link or email attachment.

Adding a new field in future (e.g. "Nominee")

Easiest path: upload this file to Claude and say "add a field called X to my card wallet" — existing data carries over untouched. To do it by hand, open the file in a text editor and make five small edits, using the existing nameOnCard field as the pattern to copy:

  1. Form — in the showCardForm function, add one fld("Label","yourField","text",c.yourField) line where you want it to appear.
  2. Save — in saveCardForm, add yourField: (f.get("yourField")||"").trim(), to the rec object.
  3. Detail view — in showCard, add a detailRow("Label", c.yourField) line.
  4. CSV — in exportCSV, add the label to the header array and c.yourField to the row array, in the same position.
  5. Search (optional) — in cardMatches, add c.yourField to the list of searched values.

Old cards without the new field simply show nothing for it — no migration needed. Test by adding one card, saving, and reopening the downloaded file before replacing your master copy.
